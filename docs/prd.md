# PRD: Football AI Prediction Platform

## 1. Objective

Build a production-grade football prediction and odds analysis platform. The platform supports general football competitions; the World Cup is one competition configuration, not a hard-coded product boundary.

The product helps football fans and operators understand match forecasts through model probabilities, likely scorelines, odds-implied probabilities, historical backtests, and plain-language explanations.

The product is not a betting execution system. It must not provide actual bet placement, stake sizing, bankroll allocation, guaranteed-profit claims, or sportsbook routing.

## 2. Target Users

### Public users

- Browse competitions, teams, fixtures, and predictions.
- View predicted scorelines, win/draw/loss probabilities, confidence ranges, model explanations, and historical model performance.
- Compare model probabilities with market-implied odds when odds data is available.

### Administrators

- Import teams, competitions, fixtures, results, and odds through CSV.
- Trigger predictions and backtests.
- Manage model versions and inspect prediction inputs.
- Review data quality issues and model evaluation reports.

There are no normal registered user accounts in the first version.

## 3. Product Scope

### V1

- Public competition and match listing pages.
- Match detail page with:
  - Predicted score.
  - Full score probability distribution.
  - Win/draw/loss probability.
  - Over/under style probability fields as analysis outputs.
  - Model explanation generated by LLM or template fallback.
  - Odds-implied probability comparison when odds data exists.
- Admin dashboard with:
  - CSV import for core entities.
  - Data quality summary.
  - Manual prediction trigger.
  - Model version list.
- Model evaluation page with:
  - Backtest results.
  - Brier Score.
  - Calibration summary.
  - Prediction accuracy by competition, model version, and date range.

### V1.1

- Real-time or near-real-time odds monitoring dashboard.
- Odds movement history.
- Market-implied probability movement.
- Alerts for large probability deviation between model and market.

### V1.2

- Model training experiment workbench.
- Feature set comparison.
- Train/evaluate/promote model workflow.
- Experiment history and reproducibility metadata.

## 4. Explicit Non-Goals

- No automatic betting.
- No bet placement instructions.
- No stake sizing, bankroll strategy, Kelly sizing, or portfolio allocation advice.
- No affiliate sportsbook links or betting funnel.
- No claims of guaranteed profit.
- No ordinary public user accounts in V1.
- No Redis, Postgres, or distributed job queue in V1.

## 5. Compliance and Safety Boundaries

- Always label predictions as probabilistic analysis.
- Always show model limitations and data quality caveats.
- Odds comparison is for market analysis only.
- Any "value deviation" language must avoid direct betting instructions.
- Public pages must include a clear disclaimer that content is not financial, betting, or investment advice.
- Admin-only research outputs must still avoid direct live betting instructions.

## 6. Tech Stack

- Web application: Next.js + TypeScript.
- Database: SQLite for V1 local-first storage.
- ORM: Prisma.
- Model service: Python.
- Prediction methods:
  - Statistical baseline: Elo, team strength, Poisson/Skellam score distribution.
  - Machine learning: tree-based model or scikit-learn-compatible first implementation.
  - Ensemble: configurable model weighting.
  - Explanation: OpenAI-compatible API and Ollama support, with template fallback.
- Background work in V1:
  - Admin-triggered jobs.
  - Local scripts.
  - Optional OS cron or scheduled command outside the app.

## 8. Success Criteria

### V1 is done when:

- A local developer can start the app and model service from documented commands.
- SQLite schema stores competitions, teams, fixtures, results, odds snapshots, predictions, model versions, and backtests.
- Admin can import seed CSV files.
- Admin can trigger predictions for fixtures.
- Public match pages show probabilities, predicted score, explanation, and odds comparison.
- Model evaluation page shows at least Brier Score and calibration summary.
- LLM explanation works through OpenAI-compatible API, Ollama, or template fallback.
- Tests cover core prediction math, probability normalization, CSV import validation, and main API routes.
- Build, lint, typecheck, and targeted model tests pass.

### V1.1 is done when:

- Odds snapshots can be ingested repeatedly.
- Dashboard shows odds movement and implied probability history.
- Dashboard flags large model-vs-market deviations without issuing bet instructions.

### V1.2 is done when:

- Admin can run model experiments from saved feature sets.
- Experiments persist parameters, data window, metrics, and artifacts.
- A model version can be promoted for prediction use.
